{%- comment -%} Single data-archive card. Rendered by _layouts/archives.html (the /data/ and /analysis/ pages) via `{% include card.html project=project %}` for each matching entry in _data/projects.yml. (The home page no longer shows a project grid.) Each project entry in _data/projects.yml provides: - title card heading - type small uppercase eyebrow (e.g. "FOIA ARCHIVE") - byline optional author/credit line; rendered in italic between the title and description (used for theses) - description 1-2 sentence summary - image optional preview thumbnail (usually /SIBLING_REPO/example-1.png). Omit for cards without a graphical preview (e.g. thesis cards). - image_position optional; set to "top" to anchor the object-cover crop to the top of the image (used for portrait document-page thumbnails, where the letterhead and title sit at the top). Default crop is centered. - links array of { title, url } pairs; the first link is also used as the card-image and title hyperlink In production the `image` URLs resolve under sustainable-fsa.com (the sibling repos are deployed to the same parent domain). They 404 in local dev — that's expected. {%- endcomment -%} {% assign project = include.project %} {% assign primary = project.links | first %}
