{%- comment -%} Single data-archive card. Rendered by _layouts/archives.html (the /data/ and /analysis/ pages) via `{% include card.html project=project %}` for each matching entry in _data/projects.yml. (The home page no longer shows a project grid.) Each project entry in _data/projects.yml provides: - title card heading - type small uppercase eyebrow (e.g. "FOIA ARCHIVE") - byline optional author/credit line; rendered in italic between the title and description (used for theses) - updated_method optional; how to resolve the archive's data - updated_ref currency date ("usdm-map", "github-file", or "manifest-dates" — see _data/projects.yml for the ref each method expects). Renders an italic "Data current as of {date}" (or, for github-file, "Data updated {date}") line in the byline slot, filled client-side by assets/archive-updated.js — hidden until the lookup succeeds, so it degrades to nothing offline. Static snapshots use a plain byline instead. - description 1-2 sentence summary - image optional preview thumbnail (usually /SIBLING_REPO/example-1.png). Omit for cards without a graphical preview (e.g. thesis cards). Shown whole (object-contain), letterboxed on the band behind it — never cropped. - image_background optional CSS color for the band behind the image. Set to "white" for figures exported on white paper/plot backgrounds so the letterbox bands disappear. Emitted inside a style attribute — plain color values only (scripts/validate-projects.mjs enforces this). Default band: ochre-light for documents cards (a manila backdrop that offsets the white page-1 thumbnails — cream is too close to white to read), cream everywhere else. - links array of { title, url } pairs; the first link is also used as the card-image and title hyperlink In production the `image` URLs resolve under sustainable-fsa.com (the sibling repos are deployed to the same parent domain). They 404 in local dev — that's expected. {%- endcomment -%} {% assign project = include.project %} {% assign primary = project.links | first %}
