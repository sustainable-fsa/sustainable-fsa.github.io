{%- comment -%} One home-page feature card. Looped from _layouts/home.html over site.data.home_cards: `{% include feature-card.html card=card %}`. include.card provides: url, label, cta, theme, icon (see _data/home_cards.yml). include.extra_class is an optional grid-placement class the caller adds to this card's outer element (home.html uses it to center a lone trailing card). The subtitle is read live from the target page's `description:` front matter, unless the card provides its own `description` (needed for external targets, which have no page front matter). Cards with `external: true` link out verbatim in a new tab instead of being prefixed with site.baseurl. The theme -> accent-class mapping is kept HERE (not in the data file) so the literal Tailwind class names stay in a file the content scanner reads; classes living only in _data/ would be purged from the production CSS. {%- endcomment -%} {%- assign card = include.card -%} {%- assign target = site.pages | where: "url", card.url | first -%} {%- case card.theme -%} {%- when "sage" -%}{%- assign hover = "hover:border-sage/40" -%}{%- assign iconbox = "bg-sage/10 text-sage-dark" -%} {%- when "ochre" -%}{%- assign hover = "hover:border-ochre/60" -%}{%- assign iconbox = "bg-ochre/10 text-ochre-dark" -%} {%- else -%}{%- assign hover = "hover:border-terracotta/40" -%}{%- assign iconbox = "bg-terracotta/10 text-terracotta" -%} {%- endcase -%} {%- comment -%} href/target are assembled outside the tag so html-validate (which cannot parse Liquid tags between attributes) sees plain attributes. Internal links get an explicit target="_self"; rel="noopener noreferrer" is harmless on internal links and required on external. {%- endcomment -%} {%- if card.external -%} {%- assign card_href = card.url -%} {%- assign card_target = "_blank" -%} {%- else -%} {%- capture card_href -%}{{ site.baseurl }}{{ card.url }}{%- endcapture -%} {%- assign card_target = "_self" -%} {%- endif -%} {%- comment -%} The class list is captured too: html-validate parses the class attribute's value, and two Liquid interpolations inside it read as duplicated "{{" / "}}" class tokens (no-dup-class). Building the string here leaves a single clean {{ card_class }} in the tag. All class tokens are still literal text in this file, so Tailwind's content scan keeps them. {%- endcomment -%} {%- capture card_class -%} group flex flex-col bg-white rounded-xl border border-gray-200 shadow-sm hover:shadow-md {{ hover }} transition-all p-6 {{ include.extra_class }} {%- endcapture -%} {%- assign card_class = card_class | strip -%}
